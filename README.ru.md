# OpenCode Client для VS Code

**English** | [Русский](README.ru.md)

---

Расширение для VS Code, которое использует локально установленный [OpenCode](https://opencode.ai) как бэкенд и предоставляет собственный нативный чат-интерфейс внутри редактора — лёгкий фронтенд для бэкенда OpenCode с первой классовой интеграцией редактора.

![Chat](docs/img/screen-1.jpg)

## Зачем

OpenCode — мощный AI-агент для кода, но его собственный TUI не является полноценной средой редактирования: он не может заменить файловый проводник VS Code, работу с git, мультикурсорное редактирование, языковые серверы, отладчик и тысячи расширений, на которые вы уже завязаны.

Существующие способы использовать OpenCode из VS Code идут с компромиссами:

- **Встроить веб-интерфейс OpenCode в панель.** Этот веб-интерфейс проектировался под браузер, а не под расширение. Он дублирует то, что VS Code уже умеет (файловое дерево, терминал), часто плодит *второй* терминал и лишён хуков интеграции, которые нужны расширению.
- **Отразить TUI-консоль в сайдбаре.** Получаете два вида одной и той же вещи и ноль нативной интеграции — вы по сути управляете терминалом внутри редактора.

Это расширение идёт другим путём: **целенаправленно построенный фронтенд, который общается с `opencode serve` через HTTP/SSE и на этом останавливается.** Никакого дублирующего файлового дерева, никакого второго терминала, никакого встроенного TUI. Каждая фича отрисована как нативный UI VS Code — диффы открываются в настоящем diff-редакторе, вопросы всплывают оверлеями, настройки живут в стандартной панели настроек.

## Возможности

**Чат и сессии**
- Стриминг ответов через SSE с рендерингом markdown (заголовки, списки, цитаты, блоки кода с подсветкой языка).
- Список сессий с поиском, закреплением, переименованием, удалением, метаданными по стоимости/токенам, с группировкой по дням.
- Переключение режимов build / plan, слеш-команды и шорткаты для скиллов.
- Черновики на сессию, повтор, инлайн-редактирование и переотправка, копирование (с фолбэком на host-clipboard — `navigator.clipboard` в webview ненадёжен).

**Подтверждения и вопросы**
- Инлайн-подтверждения вызовов инструментов (один раз / всегда / отказать).
- Клиентский **режим YOLO** — авто-подтверждение каждого вызова (`once` на запрос, без постоянных серверных грантов).
- Оверлей блокирующих вопросов от агента.

![Question](docs/img/question.jpg)

**Диффы и изменения файлов**
- Список ожидающих изменений со статистикой строк `+x/-y`, индикаторами статуса, нативными файловыми иконками VS Code.
- Клик открывает настоящий diff-редактор VS Code (не приблизительную копию в панели).
- Диффы вызовов инструментов (write/edit/str_replace) отображаются инлайн со строками `+`/`-`.

![Diffs](docs/img/diffs.jpg)

**Модели и агенты**
- С поиском пикер моделей, сгруппированный по провайдерам, с бейджами reasoning / tools / attachments и context-limit на hover.
- Скрытие целых провайдеров или отдельных моделей; активный провайдер всегда остаётся видимым.
- Представление субагентов, переключение агентов, сохраняемый выбор.

![Subagents](docs/img/subagents-view.jpg)

**Контекст и инспектор**
- Прикрепление активного файла, выделения текста, диагностики; `@mention` файлов через поиск.
- Панель инспектора: использование контекста, чек-лист todo, изменения файлов, статус MCP/LSP, скиллы.

**Прочее**
- Горячие клавиши (`⌘/Ctrl+Enter` отправка, `⌘/Ctrl+L` фокус на ввод, `⌘/Ctrl+K` новая сессия, `⌘/Ctrl+Shift+S` переключение сайдбара, `?` справка, `Esc` для прерывания занятой сессии).
- Опциональные звуковые уведомления для блокирующих событий и завершения todo.
- Учет рабочего пространства: авто-определение текущего workspace, поддержка multi-root.

![Settings](docs/img/settings.jpg)

## Требования

- [OpenCode](https://opencode.ai) установлен и доступен в PATH (`which opencode`)
- VS Code 1.85+
- Node.js 18+ (для сборки из исходников)

## Начало работы

### Установка

**Вариант A — готовый vsix (рекомендуется)**

Скачайте свежий `.vsix` со страницы [Releases](https://github.com/alexup71rus/opencode-for-vscode-light/releases), затем:

```bash
code --install-extension opencode-vscode-client-<version>.vsix
```

**Вариант B — из исходников**

```bash
npm install
npm run compile
npm run dev:install      # compile → package → установка в VS Code, затем перезагрузить окно
```

Либо собрать vsix для распространения:

```bash
npm run package:vsix     # создаёт opencode-vscode-client-<version>.vsix
```

### Использование

1. Откройте папку в VS Code.
2. Выполните команду **"OpenCode: Open Chat Panel"** из палитры команд.
3. Расширение запускает `opencode serve --port 0 --hostname 127.0.0.1` как дочерний процесс, автоматически генерирует случайный пароль и подключается.

## Конфигурация

| Настройка | По умолчанию | Описание |
|---|---|---|
| `opencode.binaryPath` | авто-определение | Путь к бинарнику opencode. Пусто — авто-определение через PATH. |
| `opencode.externalServerUrl` | (пусто) | Подключение к внешнему серверу opencode вместо запуска своего. Пример: `http://127.0.0.1:4096` |
| `opencode.serverPassword` | (пусто) | Пароль для внешнего сервера. Для запускаемых серверов генерируется автоматически. |
| `opencode.defaultModel` | (пусто) | Модель по умолчанию в формате `provider/model`. Пример: `anthropic/claude-sonnet-4-20250514` |
| `opencode.serverHostname` | `127.0.0.1` | Хостнейм для запускаемого сервера. |

## Архитектура

Extension Host крутит стек `bridge → services → protocol` и общается с webview через `postMessage`:

- **`src/extension/`** — интеграция с VS Code: активация, команды, панель webview, context provider, diff provider.
- **`src/services/`** — независимое от фреймворков состояние (сессии, модели, статистика) на небольшом EventEmitter.
- **`src/bridge/`** — подключение к OpenCode: lifecycle сервера, обёртка над SDK-клиентом, SSE-поток событий. Полностью изолирован — заменяем (HTTP → WebSocket → ACP) без затрагивания services и UI.
- **`src/protocol/`** — типизированный `postMessage` контракт. Чистые типы, ноль зависимостей.
- **`webview/src/`** — React UI, отрисовываемый внутри VS Code.

Расширение запускает `opencode serve`, парсит назначенный порт из stdout, аутентифицируется через HTTP Basic с помощью `OPENCODE_SERVER_PASSWORD` и подписывается на SSE-поток событий для обновлений в реальном времени (дельты сообщений, разрешения, изменения файлов). Дочерний процесс убивается при деактивации.

## Разработка

Инструкции по сборке, вотчерам, упаковке и dev-циклу лежат в [`docs/RUNBOOK.md`](./docs/RUNBOOK.md). Кратко:

```bash
npm run lint              # тайпчек обоих проектов (запускать перед завершением работы)
npm run compile           # разовая сборка extension + webview → dist/
npm run watch:extension   # esbuild watch (запускать вместе с watch:webview)
npm run watch:webview     # vite build watch
```

Для быстрой итерации нажмите **F5** — откроется Extension Development Host, который загружает расширение прямо из `dist/` без переустановки. Полный workflow с двумя циклами и подводными камнями — в RUNBOOK.

## Как происходит подключение к OpenCode

1. Определяет бинарник `opencode` (из настроек или через `which opencode`).
2. Запускает `opencode serve --port 0 --hostname 127.0.0.1`.
3. Парсит stdout на предмет назначенного порта.
4. Аутентификация: случайный UUID-пароль через env-переменную `OPENCODE_SERVER_PASSWORD`, HTTP Basic auth.
5. Создаёт клиент `@opencode-ai/sdk`, указывая на локальный сервер.
6. Подписывается на SSE-поток событий.
7. Убивает дочерний процесс при деактивации.
